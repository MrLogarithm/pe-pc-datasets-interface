Proto-Elamite and proto-cuneiform n-gram counts

This is an interface for exploring the n-gram counts hosted at pe-pc-datasets.

Explanation of parameters:

  • n: The length of n-gram to display. n = 1 will show the frequency of individual signs; n = 2 will show frequencies for strings of two signs; etc.
  • Exclude corrections? If checked, corrected signs (annotated with < and > in the original transcriptions) will not be counted.
  • Collapse variants? If checked, sign variants will be counted as instances of their corresponding basic sign, e.g., M066~A will be counted as an instance of M066.
  • Search string: if a search term is entered, the plot will only display counts for n-grams which contain the specified term.
  • Plot size: how many columns to display. n-grams are displayed in order of decreasing frequency. Chart.js breaks when the plots get too large, so plot size is capped at 1000.
  • Corpus: for proto-cuneiform, specify whether to return frequencies from Uruk III texts or Uruk IV texts. Both corpora contain both administrative and lexical texts.
  • Split compounds: separate sign compounds into their constituent parts before counting sign frequencies, so that e.g. |GI&GI| counts as two instances of GI rather than one instance of |GI&GI|
  • Ignore order? If checked, the order of signs in the proto-cuneiform transcriptions is ignored when counting co-occurences, so that e.g. DU EN~A PAP~A is counted as containing an instance of the "bigram" DU PAP~A.
