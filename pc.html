This interface presents n-gram counts derived from a copy of the CDLI Uruk corpus retrieved on 19 May 2020. The counts are computed on-the-fly based on your settings in the sidebar. The code used to derive the counts is available here.

Explanation of parameters:

  • Corpora: specifies which periods to include in the counts.
  • Length: The length of n-gram to display. Length = 1 will show the frequency of individual signs; Length = 2 will show strings of two signs; etc.
  • Split compounds: separate sign compounds into their constituent parts before counting sign frequencies, so that e.g. |GI&GI| counts as two instances of GI rather than one instance of |GI&GI|
  • Collapse variants? If checked, sign variants will be counted as instances of their corresponding basic sign, e.g., KUR~A will be counted as an instance of KUR.
  • Preserve order? By default, the order of signs in the transcription is ignored, so that e.g. DU EN~A PAP~A is taken to contain an instance of the bigram DU PAP~A (which, in the unordered case, is equivalent to the bigram PAP~A DU, but to avoid ambiguity this will always be printed in alphabetic order as DU PAP~A).
    If this box is checked then the order of signs in the transcription is respected, so DU EN~A is considered distinct from EN~A DU, and neither is taken to occur in the string DU EN~A PAP~A.
  • Search string: if a search term is entered, the plot will only display n-grams which contain the search string.
  • Plot size: how many rows to display. n-grams are displayed in order of decreasing frequency. Chart.js breaks when the plot gets too large, so plot size is capped at 1000.
